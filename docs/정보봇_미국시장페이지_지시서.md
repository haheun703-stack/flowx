## 0. 배경

현재 미국 시장 페이지(`/market/us`)는 `us_market_daily` 테이블 1개만 사용 중.
한국 시장 페이지(`/market`)는 **30일 차트 + 투자자 수급 + 시장체온 + 캘린더** 등 풍부한 데이터를 보여줌.
미국 시장도 같은 수준으로 끌어올리기 위해 **정보봇이 추가 수집해야 할 데이터**를 정리한 지시서.

## 2. 추가 수집 요청 — 우선순위 P0 (필수)

### 2-1. `us_market_daily` 컬럼 확장

기존 `us_market_daily` 테이블에 **JSONB 컬럼 추가**로 확장. 새 테이블 생성 불필요.

#### 추가할 JSONB 컬럼들:

```sql
ALTER TABLE us_market_daily
  ADD COLUMN IF NOT EXISTS mag7 jsonb DEFAULT NULL,
  ADD COLUMN IF NOT EXISTS futures jsonb DEFAULT NULL,
  ADD COLUMN IF NOT EXISTS crypto jsonb DEFAULT NULL,
  ADD COLUMN IF NOT EXISTS forex jsonb DEFAULT NULL,
  ADD COLUMN IF NOT EXISTS yield_curve jsonb DEFAULT NULL,
  ADD COLUMN IF NOT EXISTS breadth jsonb DEFAULT NULL,
  ADD COLUMN IF NOT EXISTS us_news jsonb DEFAULT NULL;
```

> **참고**: API route(`/api/us-market/daily/route.ts`)는 이미 `mag7`, `futures`, `crypto`, `forex`, `yield_curve`, `us_news` 컬럼을 `?? null`로 읽고 있음. 정보봇이 데이터만 넣으면 즉시 표시 가능.

#### (A) `mag7` — 매그니피센트 7 개별 주가

```json
{
  "AAPL":  { "close": 185.50, "change_pct": 1.2, "volume_ratio": 1.05 },
  "MSFT":  { "close": 420.30, "change_pct": -0.5, "volume_ratio": 0.92 },
  "GOOGL": { "close": 175.80, "change_pct": 0.8, "volume_ratio": 1.1 },
  "AMZN":  { "close": 195.40, "change_pct": 2.1, "volume_ratio": 1.3 },
  "NVDA":  { "close": 890.50, "change_pct": 3.5, "volume_ratio": 1.8 },
  "META":  { "close": 510.20, "change_pct": -1.0, "volume_ratio": 0.85 },
  "TSLA":  { "close": 175.60, "change_pct": -2.3, "volume_ratio": 1.4 }
}
```

| 필드 | 설명 |
|------|------|
| close | 종가 ($) |
| change_pct | 등락률 % |
| volume_ratio | 거래량 비율 (20일 평균 대비, 1.0 = 평균) |

**수집 소스**: Yahoo Finance API 또는 yfinance 라이브러리
**수집 시점**: 매일 미장 마감 후 (한국시간 06:00~07:00)

## 3. 추가 수집 요청 — P1 (한국시장 페이지 대비 부족분)

### 3-1. 신규 테이블: `us_market_calendar`

한국시장의 `market_calendar` 테이블에 대응하는 미국시장 이벤트 캘린더.

```sql
CREATE TABLE IF NOT EXISTS us_market_calendar (
  id         bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  date       date        NOT NULL,
  type       text        NOT NULL,   -- 'fomc' | 'earnings' | 'economic' | 'ipo' | 'expiry' | 'holiday'
  title      text        NOT NULL,
  detail     text        DEFAULT NULL,
  impact     text        DEFAULT 'medium',  -- 'high' | 'medium' | 'low'
  market     text        DEFAULT 'US',
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_us_market_calendar_date ON us_market_calendar(date);
CREATE INDEX IF NOT EXISTS idx_us_market_calendar_type ON us_market_calendar(type);
```

#### 수집할 이벤트 유형:

| type | 설명 | 소스 | 예시 |
|------|------|------|------|
| `fomc` | FOMC 회의, 의사록, 연설 | Fed 캘린더 | "FOMC 금리 결정", "파월 의장 연설" |
| `earnings` | 주요 기업 실적 발표 | Earnings Calendar | "AAPL Q1 실적", "NVDA Q1 실적" |
| `economic` | 경제지표 발표 | Investing.com 캘린더 | "CPI 발표", "고용보고서", "PMI" |
| `ipo` | 주요 IPO | IPO 캘린더 | "Reddit IPO" |
| `expiry` | 옵션/선물 만기 | | "트리플 위칭데이" |
| `holiday` | 미국 공휴일 (휴장) | | "독립기념일", "추수감사절" |

**수집 방법**:
- 월 1회 향후 60일치 이벤트를 일괄 수집 (시딩)
- 매일 D-7 이내 이벤트 업데이트 (상세 추가)

**수집 시점**: 매월 1일 + 매일 06:30 KST (미장 마감 후)

---

### 3-2. 신규 테이블: `us_investor_flow`

한국시장의 `market_investor_trend`에 대응. 미국은 외국인/기관 구분 대신 **ETF 자금 유출입**으로 대체.

```sql
CREATE TABLE IF NOT EXISTS us_investor_flow (
  date       date    NOT NULL,
  etf_ticker text    NOT NULL,    -- 'SPY' | 'QQQ' | 'IWM' | 'TLT' | 'GLD' | 'SOXX'
  flow_mil   numeric DEFAULT 0,   -- 순유입 (백만 달러, +유입/-유출)
  aum_bil    numeric DEFAULT NULL, -- 총 자산규모 (십억 달러)
  updated_at timestamptz DEFAULT now(),
  PRIMARY KEY (date, etf_ticker)
);
```

| 필드 | 설명 |
|------|------|
| date | 날짜 |
| etf_ticker | ETF 종목코드 |
| flow_mil | 일일 순유출입 (백만 달러) |
| aum_bil | 운용자산 (십억 달러) |

**대상 ETF** (6개): `SPY`, `QQQ`, `IWM`, `TLT`, `GLD`, `SOXX`
**수집 소스**: ETF.com, etfdb.com, Bloomberg (30일치 유지)
**수집 시점**: 매일 07:00 KST

---

### 3-3. 30일 히스토리 (차트용)

한국시장 페이지는 30일 일봉 차트를 보여줌. 미국시장도 동일하게 구현하려면 `us_market_daily`에 **최근 30영업일** 데이터가 필요.

**현재 상태**: `us_market_daily`가 매일 1행씩 쌓이고 있다면, 이미 30일치가 존재할 수 있음.
**확인 필요**: 정보봇이 매일 갱신할 때 `UPSERT`로 처리하고 있는지? 아니면 최신 1행만 유지하는지?

> **중요**: 반드시 `INSERT ... ON CONFLICT (date) DO UPDATE` 패턴을 사용하여 날짜별로 행이 쌓이도록 할 것. `DELETE` 후 `INSERT`는 금지.

## 5. 웹봇 후속 작업 (정보봇 수집 완료 후)

정보봇이 데이터 수집을 시작하면, 웹봇이 수행할 작업:

1. **MAG7 패널 구현** — `mag7` JSONB 데이터를 7개 카드로 표시
2. **시장 폭(Breadth) 패널** — 상승/하락 비율 게이지, 52주 신고/신저가
3. **국채 금리 커브 차트** — `yield_curve` 데이터를 꺾은선 차트로
4. **ETF 자금흐름 차트** — `us_investor_flow` 30일 막대 차트
5. **미국 경제 캘린더** — `us_market_calendar` → `MarketCalendarPanel` 재사용
6. **환율/암호화폐 미니 패널** — `forex`, `crypto`
7. **뉴스 헤드라인 패널** — `us_news` 상위 5개
8. **30일 지수 차트** — `us_market_daily` 30행 → lightweight-charts
