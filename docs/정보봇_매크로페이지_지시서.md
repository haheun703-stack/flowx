## 3. `macro_data` 테이블 상세

### 스키마 (JSONB 문서형)

```sql
CREATE TABLE macro_data (
  id            TEXT PRIMARY KEY,   -- 'macro-YYYY-MM-DD'
  date          DATE NOT NULL,
  indices       JSONB,              -- 글로벌 지수 배열
  commodities   JSONB,              -- 원자재 배열
  forex         JSONB,              -- 환율 배열
  crypto        JSONB,              -- 크립토 배열
  bonds         JSONB,              -- 채권 금리 배열
  vix           JSONB,              -- VIX 단일 객체
  updated_at    TIMESTAMPTZ
);
```

### 각 JSONB 필드의 아이템 구조

```json
{ "symbol": "SPX", "name": "S&P 500", "price": 5123.45, "change": 1.23 }
```

> `price` = 현재 가격/값, `change` = 전일 대비 변동률 (%)

### 필수 심볼 목록 (총 21개)

| 카테고리 | JSONB 필드 | 심볼 목록 | 웹 표시명 |
|---------|-----------|----------|----------|
| **지수** (8개) | `indices` | SPX, IXIC, DJI, N225, HSI, GDAXI, SSEC, FTSE | S&P 500, NASDAQ, 다우존스, 닛케이225, 항셍, DAX, 상해종합, FTSE100 |
| **원자재** (4개) | `commodities` | WTI, GOLD, SILVER, COPPER | WTI유, 금, 은, 구리 |
| **환율** (4개) | `forex` | USDKRW, DXY, USDJPY, EURUSD | 원/달러, 달러인덱스, 엔/달러, 유로/달러 |
| **크립토** (3개) | `crypto` | BTC, ETH, SOL | 비트코인, 이더리움, 솔라나 |
| **채권** (2개) | `bonds` | US10Y, US2Y | 미국 10년물, 미국 2년물 |
| **VIX** (1개) | `vix` | — (단일 객체) | VIX |

### VIX 필드 구조 (다른 필드와 다름)

```json
// vix는 배열이 아닌 단일 객체
"vix": { "price": 18.5, "change": 0.25 }
```

### 경고 임계값 (프론트 하드코딩)

| 심볼 | 임계값 | 방향 | 경고 조건 |
|------|--------|------|----------|
| VIX | 25 | above | `vix.price ≥ 25` → 빨간 경고 |
| USDKRW | 1400 | above | `forex.USDKRW.price ≥ 1400` → 빨간 경고 |

### FearGreedGauge에서 사용하는 심볼

| 심볼 | 역할 | 표시 |
|------|------|------|
| `FNG` 또는 `FEAR_GREED` | 공포탐욕지수 (0~100) | 반원 게이지 |
| `VIX` | 변동성 지수 | 하단 행 (≥25 빨간 경고) |

> FNG는 `indices` 또는 별도 필드에 포함해야 함. 현재 `sentiment` 카테고리로 분류됨.
> 프론트에서 `FNG` 또는 `FEAR_GREED` 심볼을 찾음 → **둘 중 하나는 반드시 포함**

### 갱신 규칙

- **담당**: ⭐ **정보봇** (`scheduled_macro_daily_record`)
- **갱신 시각**: 매일 **16:25**
- **갱신 방식**: id = `macro-YYYY-MM-DD`로 upsert
- **중요**: 모든 JSONB 필드에 위 21개 심볼이 빠짐없이 포함되어야 함

## 4. `cost_floor` 테이블 상세

### 스키마

```sql
CREATE TABLE cost_floor (
  name           TEXT PRIMARY KEY,  -- 표시 이름 (WTI, Gold 등)
  category       TEXT,              -- energy, metal, grain 등
  ticker         TEXT,              -- 야후 파이낸스 티커 (CL=F, GC=F 등)
  floor_name     TEXT,              -- 지지선 이름
  floor_price    NUMERIC,           -- 지지선 가격
  ceiling_name   TEXT,              -- 저항선 이름
  ceiling_price  NUMERIC,           -- 저항선 가격
  current_price  NUMERIC,           -- 현재 가격
  position_pct   NUMERIC,           -- 현재 위치 % (0~100)
  change_pct     NUMERIC,           -- 전일 대비 등락률 (%)
  sector         TEXT,              -- 섹터 분류 (선택)
  catalyst       TEXT,              -- 참고 노트 (선택)
  updated_at     TIMESTAMPTZ
);
```

### 프론트에서의 단위 매핑

| ticker | 단위 | 표시 |
|--------|------|------|
| CL=F | USD/bbl | WTI 원유 |
| BZ=F | USD/bbl | 브렌트유 |
| NG=F | USD/MMBtu | 천연가스 |
| GC=F | USD/oz | 금 |
| SI=F | USD/oz | 은 |
| HG=F | USD/lb | 구리 |
| ZC=F | USd/bu | 옥수수 |
| ZW=F | USd/bu | 밀 |
| ZS=F | USd/bu | 대두 |

### 게이지 색상 기준 (position_pct 기반)

| position_pct | 색상 | 의미 |
|-------------|------|------|
| 0~15% | 🟢 녹색 | 바닥 근접 ("바닥 근접" 경고) |
| 16~20% | 🟢 녹색 | 지지선 부근 |
| 21~40% | 🟡 연두 | 하단 영역 |
| 41~60% | 🟡 노랑 | 중간 |
| 61~80% | 🟠 주황 | 상단 영역 |
| 80~100% | 🔴 빨강 | 천장 근접 ("천장 근접" 경고) |

### position_pct 계산 공식

```python
position_pct = (current_price - floor_price) / (ceiling_price - floor_price) * 100
```

### 갱신 규칙

- **담당**: ⭐ **정보봇**
- **갱신 시각**: 별도 스케줄 (원자재 시장 마감 후)
- **갱신 방식**: name 기준 upsert
- **중요 사항**:
  - `current_price`와 `position_pct`는 매일 갱신
  - `floor_price`, `ceiling_price`는 기술 분석 기반으로 주기적 갱신 (주 1회 이상)
  - `position_pct`는 반드시 0~100 범위 (음수/100 초과 허용하지만 게이지가 넘침)

## 6. 갱신 검증 체크리스트

```
□ macro_data — id = 'macro-오늘날짜' 존재
  ├ indices 배열: 8개 심볼 (SPX, IXIC, DJI, N225, HSI, GDAXI, SSEC, FTSE)
  ├ commodities 배열: 4개 심볼 (WTI, GOLD, SILVER, COPPER)
  ├ forex 배열: 4개 심볼 (USDKRW, DXY, USDJPY, EURUSD)
  ├ crypto 배열: 3개 심볼 (BTC, ETH, SOL)
  ├ bonds 배열: 2개 심볼 (US10Y, US2Y)
  ├ vix 객체: price > 0
  └ 모든 아이템의 price > 0, change는 숫자

□ macro_data — FNG/FEAR_GREED 심볼이 indices 또는 별도 필드에 존재
  └ FearGreedGauge가 이 심볼을 검색함 → 없으면 게이지 빈칸

□ cost_floor — 행 수 ≥ 5 (최소 주요 원자재)
  ├ 모든 행의 current_price > 0
  ├ 모든 행의 position_pct BETWEEN 0 AND 100 (권장)
  ├ floor_price < ceiling_price (논리적 일관성)
  └ updated_at이 최근 3일 이내
```

## 8. 주의 사항

1. **FNG/FEAR_GREED 심볼 필수**: `FearGreedGauge`가 `macro_data` 응답에서 `FNG` 또는 `FEAR_GREED` 심볼을 검색함. 없으면 게이지가 빈칸으로 표시됨. **반드시 포함**.

2. **VIX는 별도 구조**: `vix`는 배열이 아닌 `{ price, change }` 단일 객체. 다른 JSONB 필드들과 구조가 다름.

3. **change 필드 = 등락률(%)**: `change`는 등락 포인트가 아닌 **퍼센트**. 프론트에서 `change_pct`로 표시.

4. **cost_floor position_pct**: 정보봇이 계산해서 넣어야 함. 프론트에서는 그대로 게이지에 표시만 함.

5. **캐시**: `useMacroDaily()` 5분 stale + 10분 refetch, `useCostFloor()` 30분 stale. 갱신 후 최대 10분 뒤 프론트 반영.
