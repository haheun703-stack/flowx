## 1. [P0] macro_dashboard 자동 수집 cron 등록

### 배경
- 웹봇이 `scripts/update-macro-data.mjs` 스크립트 작성 완료
- Supabase `macro_dashboard` 테이블 생성 완료 (2026-04-26)
- `.env.local`에 `FRED_API_KEY` 등록 완료
- **하지만 cron이 없어서 데이터가 자동으로 수집되지 않음**

### 정보봇이 할 일

#### 방법 A: 정보봇 스케줄러에 태스크 추가 (권장)

정보봇 스케줄러(`src/scheduler.py`)에 아래 태스크 추가:

```python
# 거시경제 대시보드 자동 수집 (매 평일 07:00 KST)
# 미국 장 마감 후 Yahoo Finance + FRED 데이터 수집 → Supabase macro_dashboard 업서트
schedule.every().day.at("07:00").do(run_macro_dashboard_update)
```

실행할 명령:
```bash
cd d:\kr.TradingView.site_웹봇\kr.TradingView.site_웹사이트\signal-os
node scripts/update-macro-data.mjs
```

#### 방법 B: Windows 작업 스케줄러 등록


### 스크립트 상세

| 항목 | 값 |
|------|-----|
| 스크립트 경로 | `signal-os/scripts/update-macro-data.mjs` |
| 필요 환경변수 | `NEXT_PUBLIC_SUPABASE_URL`, `SUPABASE_SERVICE_KEY`, `FRED_API_KEY`(선택) |
| 대상 테이블 | `macro_dashboard` (date PK, data JSONB) |
| 데이터 소스 | Yahoo Finance (S&P500, 환율, 원유) + FRED (Fed금리, CPI) |
| 스케줄 | `0 7 * * 1-5` (매 평일 07:00 KST) |
| 출력 | `macro_dashboard` 테이블에 당일 행 upsert |

### 수동 테스트


## 2. [P1] stock_valuations STALE 해결

### 현상
- `/api/health` 헬스체크에서 `stock_valuations` 테이블이 STALE (30일 이상 미갱신)
- 2026-04-23 점검 시 확인됨

### 원인 (4/23 분석)
- 정보봇 `StockValuationScanner`가 AWS S3에서 `_KS` 폴더를 참조하는데, 해당 폴더가 존재하지 않음
- carry-forward(이전 데이터 유지)만 동작하고 신규 갱신이 안 됨

### 정보봇이 할 일
1. `StockValuationScanner` 소스 확인 — AWS S3 `_KS` 폴더 경로 수정 또는 데이터 소스 변경
2. 수동 1회 실행으로 갱신 테스트:
   ```python
   python -c "from src.infrastructure.intelligence.stock_valuation_scanner import StockValuationScanner; StockValuationScanner().scan()"
   ```
3. 정상 갱신 확인 후 스케줄(17:05) 정상 동작 확인

### 검증
```sql
SELECT COUNT(*), MAX(updated_at) FROM stock_valuations;
-- 기대: COUNT ≥ 100, MAX(updated_at) = 오늘
```

## 3. [P2] /global-economy 하드코딩 → API 자동 전환

### 현상
- `/global-economy` 페이지의 Group1~6 (28개 차트)가 모두 **하드코딩**
- 2026-04-18에 수동으로 최신화했으나, 매번 수동 업데이트는 지속 불가

### 구조
```
[정보봇 cron] → scripts/update-macro-data.mjs → Supabase macro_dashboard
                                                      ↓
[웹봇] → /api/macro/dashboard → useMacroDashboardData() hook
                                                      ↓
[프론트] → MacroDashboardView.tsx (현재 하드코딩 → hook으로 교체 필요)
```

### 정보봇이 할 일
1. **1번(cron 등록)을 먼저 완료** — 데이터가 Supabase에 쌓여야 교체 가능
2. 데이터가 쌓이면 웹봇이 프론트엔드 코드를 하드코딩 → API 호출로 교체

### 현재 수집 데이터 (update-macro-data.mjs)
- S&P 500 가격 + YTD 수익률
- USD/KRW, USD/JPY 환율
