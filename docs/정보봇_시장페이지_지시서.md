## 2. 박스별 상세 데이터 소스

### 2-1. 1행: KOSPI/KOSDAQ 30일 차트

| 구분 | API 엔드포인트 | 데이터 소스 | 담당 |
|------|---------------|------------|------|
| KOSPI 일봉 30일 | `/api/market/daily` | **네이버 주식 API** (외부) | 외부 API (갱신 불필요) |
| KOSDAQ 일봉 30일 | `/api/market/daily-kosdaq` | **네이버 주식 API** (외부) | 외부 API (갱신 불필요) |
| KOSPI 투자자 수급 | `/api/market/investor-flow` | **`market_investor_trend`** (KOSPI) | ⭐ **정보봇** |
| KOSDAQ 투자자 수급 | `/api/market/investor-flow-kosdaq` | **`market_investor_trend`** (KOSDAQ) | ⭐ **정보봇** |

#### `market_investor_trend` 테이블 스키마

```sql
-- PK: (date, market)
CREATE TABLE market_investor_trend (
  date        DATE NOT NULL,
  market      TEXT NOT NULL,      -- 'KOSPI' | 'KOSDAQ'
  foreign_net BIGINT,             -- 외국인 순매수 (백만원)
  inst_net    BIGINT,             -- 기관 순매수 (백만원)
  individual_net BIGINT,          -- 개인 순매수 (백만원)
  updated_at  TIMESTAMPTZ
);
```

**갱신 규칙**:
- **갱신 시각**: 매일 **16:47**
- **갱신 방식**: `MarketInvestorTrendScanner` → Supabase upsert
- **행 수**: 30영업일 × 2시장 = ~60행
- **단위**: **백만원** (프론트에서 `/100` → 억원 변환)
- **필수 확인**: 오늘 날짜로 KOSPI 1행 + KOSDAQ 1행 = 2행 존재

### 2-3. 3행 좌: 시장 체온

| 필드 | 데이터 소스 | 설명 |
|------|------------|------|
| `breadth` | `market_overview.breadth` | 상승 종목 비율 (0~1) |
| `stocks_up` | `market_overview.stocks_up` | 상승 종목 수 |
| `stocks_down` | `market_overview.stocks_down` | 하락 종목 수 |
| `stocks_flat` | `market_overview.stocks_flat` | 보합 종목 수 |

**시장 체온 기준**:
- ≤20%: 매우 차가움 (파랑)
- 21~40%: 차가움 (주황)
- 41~60%: 보통 (회색)
- 61~80%: 따뜻함 (빨강)
- >80%: 매우 뜨거움 (녹색)

## 3. `market_overview` 테이블 상세

> 이 테이블이 /market 페이지의 **핵심 데이터 허브** (2행~4행이 전부 의존)

### 스키마

```sql
CREATE TABLE market_overview (
  date               DATE PRIMARY KEY,
  kospi_close        NUMERIC,        -- KOSPI 종가
  kospi_change_pct   NUMERIC,        -- KOSPI 등락률 (%)
  kosdaq_close       NUMERIC,        -- KOSDAQ 종가
  kosdaq_change_pct  NUMERIC,        -- KOSDAQ 등락률 (%)
  sp500_close        NUMERIC,        -- S&P 500 종가
  sp500_change_pct   NUMERIC,        -- S&P 500 등락률 (%)
  nasdaq_close       NUMERIC,        -- 나스닥 종가
  nasdaq_change_pct  NUMERIC,        -- 나스닥 등락률 (%)
  stocks_up          INTEGER,        -- 상승 종목 수
  stocks_down        INTEGER,        -- 하락 종목 수
  stocks_flat        INTEGER,        -- 보합 종목 수
  breadth            NUMERIC,        -- 상승 비율 (0~1)
  foreign_net        BIGINT,         -- 외국인 순매수 (원)
  inst_net           BIGINT,         -- 기관 순매수 (원)
  individual_net     BIGINT,         -- 개인 순매수 (원)
  foreign_trend      TEXT            -- 외국인 추세
);
```

### 갱신 규칙
- **담당**: ⭐ **정보봇** (`MarketOverviewScanner`)
- **갱신 시각**: 매일 **16:45**
- **갱신 방식**: 오늘 날짜로 upsert (INSERT or UPDATE)
- **확인 사항**: date = 오늘, 모든 필드 NOT NULL

### 프론트에서의 단위 변환
- `foreign_net`, `inst_net`, `individual_net`: 원 → 억원 (`/ 10000` 후 formatBil)
- `breadth`: 0~1 → % (`× 100`)

## 5. 갱신 검증 체크리스트

```
□ market_overview       — date = 오늘, 모든 필드 NOT NULL
  ├ kospi_close > 0, kosdaq_close > 0
  ├ sp500_close > 0, nasdaq_close > 0
  ├ stocks_up + stocks_down + stocks_flat > 100
  ├ breadth BETWEEN 0 AND 1
  └ foreign_net, inst_net, individual_net ≠ NULL

□ market_investor_trend — date = 오늘, market = 'KOSPI' 1행 존재
□ market_investor_trend — date = 오늘, market = 'KOSDAQ' 1행 존재
  └ foreign_net, inst_net, individual_net 모두 NOT NULL

□ treemap_stocks        — 전체 행 수 ≥ 200, updated_at = 오늘
  └ market_cap > 0인 행만 사용됨
```
