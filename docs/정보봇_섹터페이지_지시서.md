## 3. 테이블별 상세

---

### 3-1. `sector_universe` — 섹터맵 + 공급망의 핵심

**사용 페이지**: `/sectors` (섹터맵), `/sectors/[key]` (공급망 분석), `/scenario` (시나리오)

#### 스키마

| 컬럼 | 타입 | 설명 | 예시 |
|------|------|------|------|
| `sector_key` | TEXT | 섹터 식별자 | `semiconductor`, `bio`, `ev` |
| `sector_name` | TEXT | 섹터 한글명 | `반도체`, `바이오`, `2차전지` |
| `stock_name` | TEXT | 종목명 (한글) | `삼성전자`, `SK하이닉스` |
| `ticker` | TEXT | 종목 코드 | `005930`, `NVDA` |
| `market` | TEXT | 시장 | `KR`, `US` |
| `tier` | INTEGER | 공급망 계층 (1~5) | `5` = ETF·글로벌대형, `1` = 소재·부품 |
| `sub_category` | TEXT | 하위 분류 | `메모리`, `파운드리`, `장비` |
| `change_pct` | NUMERIC | 등락률 (%) | `2.34`, `-1.52` |
| `foreign_net` | BIGINT | 외국인 순매수 | `50000000000` (원) |
| `institution_net` | BIGINT | 기관 순매수 | `30000000000` (원) |
| `theme_tags` | TEXT[] | 테마 태그 배열 | `['HBM', 'AI']` |
| `updated_at` | TIMESTAMPTZ | 갱신 시각 | |

#### 공급망 계층 (Tier) 정의

| Tier | 이름 | 색상 | 예시 종목 |
|------|------|------|----------|
| 5 | ETF · 글로벌 대형 | 보라 | NVDA, TSMC, SOXX |
| 4 | 글로벌 소부장 | 파랑 | ASML, LRCX, AMAT |
| 3 | 한국 대형 | 녹색 | 삼성전자, SK하이닉스 |
| 2 | 한국 소부장 | 주황 | 한미반도체, ISC |
| 1 | 소재·부품 | 노랑 | 동진쎄미켐, 에스앤에스텍 |

#### 프론트 사용 방식

**섹터맵 (`/sectors`)**:
- `sector_key` + `sector_name`으로 그룹핑
- 섹터별: 종목수, 링크수, 평균 등락률, 티어 분포, TOP 3 종목
- 13개 섹터 카드 그리드 표시

**공급망 (`/sectors/[key]`)**:
- `tier`별 수평 레인 (Swimlane)
- `sub_category`별 그룹 내 종목 카드
- 카드: 종목명 + 티커 + 등락률 (색상)
- `theme_tags`로 테마 필터 (HBM, AI 등)

#### 갱신 규칙

- **KR 종목**: 매일 **15:35** (장 마감 후) — `SectorSupplyScanner`
- **US 종목**: 매일 **06:05** (화~토, US장 마감 후) — `SectorSupplyScanner`
- **갱신 방식**: `(sector_key, ticker)` 기준 upsert
- **중요**: `change_pct`는 **당일 등락률**이므로 매일 갱신 필수
- **중요**: `foreign_net`, `institution_net`도 매일 갱신

### 3-2. `sector_links` — 공급망 연결선

**사용 페이지**: `/sectors` (섹터맵 - 링크수), `/sectors/[key]` (공급망 연결선)

#### 스키마

| 컬럼 | 타입 | 설명 | 예시 |
|------|------|------|------|
| `sector_key` | TEXT | 섹터 식별자 | `semiconductor` |
| `from_stock` | TEXT | 출발 종목 (티커) | `NVDA` |
| `to_stock` | TEXT | 도착 종목 (티커) | `005930` |
| `relation` | TEXT | 관계 유형 | `공급`, `경쟁`, `자회사`, `협력`, `고객` |
| `strength` | NUMERIC | 연결 강도 (0~1) | `0.8` |

#### 관계 유형별 색상 (프론트 표시)

| relation | 뱃지색 | 의미 |
|----------|--------|------|
| `공급` | 파랑 | A가 B에 납품 |
| `경쟁` | 빨강 | A와 B가 경쟁 |
| `자회사` | 보라 | A가 B의 자회사 |
| `협력` | 녹색 | A와 B가 협력 |
| `고객` | 주황 | A가 B의 고객 |

#### 갱신 규칙

- **갱신 주기**: **일요일 22:00** (주 1회) — `scheduled_sector_links_seed`
- **갱신 방식**: `(sector_key, from_stock, to_stock)` 기준 upsert
- **참고**: 공급망 관계는 자주 변하지 않으므로 주 1회면 충분
- **신규 종목 추가 시**: `sector_universe`에 종목 추가 → `sector_links`에 관계 추가

### 3-3. `treemap_stocks` — 트리맵 시각화

**사용 페이지**: `/market/treemap`

#### 스키마

| 컬럼 | 타입 | 설명 | 예시 |
|------|------|------|------|
| `ticker` | TEXT PK | 종목 코드 | `005930` |
| `name` | TEXT | 종목명 | `삼성전자` |
| `sector` | TEXT | 섹터명 | `반도체`, `바이오` |
| `market` | TEXT | 시장 | `KOSPI`, `KOSDAQ` |
| `change_pct` | NUMERIC | 당일 등락률 (%) | `2.34` |
| `foreign_net` | BIGINT | 외국인 순매수 | 원 단위 |
| `institution_net` | BIGINT | 기관 순매수 | 원 단위 |
| `price` | NUMERIC | 현재가 | `72000` |
| `market_cap` | BIGINT | 시가총액 | 억원 단위 |
| `trading_volume` | BIGINT | 거래대금 | 원 단위 |
| `updated_at` | TIMESTAMPTZ | 갱신 시각 | |

#### 프론트 사용 방식

- **섹터별 뷰**: sector로 그룹 → 섹터 트리맵 (22개 섹터)
- **전체종목 뷰**: 80개 선별 (섹터 비중 비례) → 캔버스 트리맵
- **크기 기준**: market_cap (기본) / trading_volume / |change_pct|
- **색상**: change_pct 기반 Finviz 7단계 (빨강~파랑)
- **필터**: 전체/KOSPI/KOSDAQ

#### 갱신 규칙

- **갱신 시각**: 매일 **16:52** — `SectorSupplyScanner`
- **갱신 방식**: `ticker` 기준 upsert
- **필수**: `market_cap > 0` (0이면 트리맵에서 제외됨)
- **필수**: 전체 행 수 ≥ 200
- **중요**: `change_pct`, `foreign_net`, `institution_net` 매일 갱신

### 3-4. `sector_leaders` — 글로벌 대장주

**사용 페이지**: `/sectors/leaders`

#### 스키마

| 컬럼 | 타입 | 설명 | 예시 |
|------|------|------|------|
| `sector` | TEXT | 섹터명 | `반도체`, `2차전지` |
| `leader_ticker` | TEXT | 대장주 티커 | `NVDA`, `TSLA` |
| `leader_name` | TEXT | 대장주 이름 | `NVIDIA`, `Tesla` |
| `leader_exchange` | TEXT | 거래소 | `NASDAQ`, `NYSE` |
| `leader_price` | NUMERIC | 대장주 현재가 (USD) | `875.50` |
| `leader_change_pct` | NUMERIC | 대장주 등락률 (%) | `2.34` |
| `leader_theme` | TEXT | 테마 태그 | `AI`, `HBM` |
| `kr_ticker` | TEXT | 한국 연관 종목 코드 | `005930` |
| `kr_name` | TEXT | 한국 연관 종목명 | `삼성전자` |
| `relation` | TEXT | 관계 유형 | `공급`, `경쟁`, `자회사`, `협력`, `고객` |
| `revenue_dependency` | TEXT | 매출 의존도 | `높음`, `중간`, `낮음` |
| `detail` | TEXT | 상세 설명 | `HBM 납품` |
| `updated_at` | TIMESTAMPTZ | 갱신 시각 | |

#### 프론트 사용 방식

- 섹터별 그룹 → 대장주 카드 (이름/티커/거래소/가격/등락률)
- 대장주 하위: 한국 연관 종목 행 (관계뱃지 + 매출의존도 + 상세)
- 섹터 필터 드롭다운

#### 매출 의존도 색상

| revenue_dependency | 색상 | 의미 |
|-------------------|------|------|
| `높음` | 빨강 | 매출 30%+ 의존 |
| `중간` | 주황 | 매출 10~30% 의존 |
| `낮음` | 녹색 | 매출 10% 미만 |

#### 갱신 규칙

- **갱신 시각**: 매일 **16:53** — `GlobalLeadersScanner`
- **갱신 방식**: `(leader_ticker, kr_ticker)` 기준 upsert
- **매일 갱신 필수**: `leader_price`, `leader_change_pct`
- **주기적 갱신**: `relation`, `revenue_dependency`, `detail` (변경 시)

## 6. 갱신 검증 체크리스트

```
□ sector_universe (KR) — updated_at = 오늘, KR 종목 수 ≥ 100
  ├ 13개 섹터 모두 종목 존재 (sector_key 고유 개수 ≥ 13)
  ├ change_pct 값이 유효 (NULL이면 0으로 표시됨)
  ├ tier BETWEEN 1 AND 5 (범위 밖이면 프론트에서 무시)
  └ sub_category NOT NULL (Swimlane 그룹핑에 필수)

□ sector_universe (US) — 화~토 06:05 갱신
  ├ market = 'US' 종목 존재
  └ change_pct 당일 미국 시장 등락률 반영

□ sector_links — 주 1회 갱신 (일요일 22:00)
  ├ 13개 섹터 모두 링크 존재
  ├ from_stock, to_stock이 sector_universe에 존재하는 티커
  └ relation ∈ {'공급', '경쟁', '자회사', '협력', '고객'}

□ treemap_stocks — 전체 행 수 ≥ 200
  ├ market_cap > 0 (0이면 트리맵에서 제외)
  ├ sector NOT NULL (NULL이면 '기타'로 분류)
  ├ change_pct 당일 등락률 반영
  └ updated_at = 오늘

□ sector_leaders — 행 수 ≥ 10
  ├ leader_price > 0 (당일 미국 시장 종가)
  ├ leader_change_pct 당일 등락률 반영
  ├ kr_ticker가 유효한 한국 종목 코드
  └ relation ∈ {'공급', '경쟁', '자회사', '협력', '고객'}

□ sector_heatmap — date = 오늘
  ├ 섹터 수 ≥ 10
  └ score 값이 유효한 숫자

□ sector_rotation — date = 오늘
  ├ rank 1부터 순차적
  └ 섹터 수 ≥ 3 (최소)
```

## 8. 주의 사항

1. **sector_universe는 가장 중요한 테이블**: 섹터맵 + 공급망 + 시나리오 페이지가 모두 의존. 갱신 실패 시 3개 페이지 동시 영향.

2. **change_pct 매일 갱신 필수**: 섹터맵의 평균 등락률, 공급망의 종목 카드 색상, 트리맵의 색상이 모두 이 값에 의존. 갱신 안 하면 어제 데이터로 표시됨.

3. **sector_links는 주 1회**: 공급망 관계(공급/경쟁/자회사)는 자주 변하지 않음. 일요일에 한 번만 시드(seed). 단, 신규 종목 추가 시 수동 갱신 필요.

4. **treemap_stocks vs sector_universe**: 둘 다 종목 데이터이지만 용도가 다름.
   - `sector_universe`: 공급망 분석용 (tier/sub_category/theme_tags 포함)
   - `treemap_stocks`: 시각화용 (market_cap/trading_volume 포함)
   - 둘 다 `change_pct`, `foreign_net`, `institution_net` 매일 갱신

5. **sector_leaders의 leader_price**: 미국 시장 종가. US 장 마감 후(06:05~) 갱신하거나, 16:53에 전일 종가 기준으로 갱신.

6. **캐시**: 모든 React Query 훅이 5분 staleTime. 갱신 후 최대 5분 뒤 프론트 반영.

7. **sector_key 유효성**: `sector_universe`에 종목 추가 시 `sector_key`가 위 13개 목록에 포함되어야 함. 없으면 `/sectors` 카드에는 표시되지만 `/sectors/[key]` 접근 불가.
